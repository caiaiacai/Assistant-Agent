# 源代码清单说明

> 软著登记提交源代码要求：**连续的前 30 页 + 连续的后 30 页**，每页 50 行，Courier New 或类似等宽字体，页眉注明软件名和版本。下面列出建议打包进 60 页源代码材料的模块顺序。

## 代码组装顺序建议

### 前 30 页（核心启动 + 调度链路）
1. `src-tauri/src/lib.rs` — 应用入口、模块注册、Tauri setup、APP_HANDLE 全局句柄
2. `src-tauri/src/main.rs` — 二进制 entrypoint
3. `src-tauri/src/db.rs` — SQLite 初始化、迁移
4. `src-tauri/src/settings.rs` — 配置 KV 存储、数据结构
5. `src-tauri/src/todo.rs` — TODO 实体、CRUD
6. `src-tauri/src/commands.rs` — Tauri 命令入口
7. `src-tauri/src/planner.rs`（开头部分）— 规划器 Prompt、Plan 结构

### 后 30 页（核心业务逻辑 + 前端）
1. `src-tauri/src/executor.rs` — Agent DAG 调度器、工具执行循环
2. `src-tauri/src/llm.rs` — LLM 调用封装
3. `src-tauri/src/tools.rs` — 工具实现（web_search/web_fetch/fs_*）
4. `src-tauri/src/reminder.rs` — 自然语言时间解析、提醒调度器
5. `src-tauri/src/location_macos.rs` — CoreLocation 原生定位桥
6. `src-tauri/src/gateway/wechat.rs` — 微信网关
7. `src/pages/Main.tsx` — 前端主界面
8. `src/pages/Settings.tsx` — 前端设置页
9. `src/api.ts` — 前端 IPC 包装

## 排版命令（待用）

使用 `enscript` 或 `a2ps` 等工具可以快速生成 PDF：

```bash
# 合并核心 Rust 源代码
cat \
  src-tauri/src/lib.rs \
  src-tauri/src/main.rs \
  src-tauri/src/db.rs \
  src-tauri/src/settings.rs \
  src-tauri/src/todo.rs \
  src-tauri/src/commands.rs \
  src-tauri/src/planner.rs \
  > /tmp/sath_front30.txt

# 后 30 页
cat \
  src-tauri/src/executor.rs \
  src-tauri/src/llm.rs \
  src-tauri/src/tools.rs \
  src-tauri/src/reminder.rs \
  src-tauri/src/location_macos.rs \
  src-tauri/src/gateway/wechat.rs \
  src/pages/Main.tsx \
  src/pages/Settings.tsx \
  src/api.ts \
  > /tmp/sath_back30.txt

# 生成 PDF（需安装 enscript + ps2pdf）
enscript -B -f "Courier9" --word-wrap -p - /tmp/sath_front30.txt | ps2pdf - sath_source_front30.pdf
enscript -B -f "Courier9" --word-wrap -p - /tmp/sath_back30.txt | ps2pdf - sath_source_back30.pdf
```

提交前再手动核验一下：
- 每页约 50 行左右
- 前 30 页和后 30 页**内容不重复**
- 代码中不含敏感信息（API key、真实账号等）
