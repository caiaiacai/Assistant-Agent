# SATH 软著申请材料包

本目录存放 SATH 软件著作权登记所需的材料，随开发进度持续更新。

## 目录结构

```
docs/copyright/
├── README.md              本文件：总索引 + 提交清单
├── 设计说明书.md           系统架构、模块设计、技术决策
├── 用户手册.md             面向最终用户的操作说明
├── 源代码清单说明.md       提交代码的组装顺序与打印方法
└── screenshots/           运行截图（待用户补充）
```

## 软著提交清单

中国版权保护中心软件著作权登记需要以下材料：

| # | 材料 | 当前状态 | 说明 |
|---|------|---------|------|
| 1 | 软件著作权登记申请表 | ⬜ 提交前填写 | 在 [中国版权保护中心](http://www.ccopyright.com.cn) 官网在线填 |
| 2 | 软件说明书（设计文档） | ✅ 已完成初稿 | [设计说明书.md](./设计说明书.md) |
| 3 | 用户操作手册 | ✅ 已完成初稿 | [用户手册.md](./用户手册.md)（可与设计说明书合并为一份"软件说明书"也行） |
| 4 | 源代码（前 30 页 + 后 30 页） | ⬜ 待排版 | 见 [源代码清单说明.md](./源代码清单说明.md) |
| 5 | 权利归属证明 | ⬜ 视情况 | 个人开发者一般不需要 |
| 6 | 身份证明复印件 | ⬜ 提交时上传 | 申请人身份证 |
| 7 | 软件运行截图 | ⬜ 待截图 | 见下方 [截图清单](#截图清单) |

## 截图清单

建议在主要功能稳定后截图存档到 `screenshots/`，用于软著说明书和提交时的"软件运行效果"。每次 UI 大改后也应更新。

**必须包含的截图**：

- [ ] `01_onboarding.png` — 首次启动引导页
- [ ] `02_main_empty.png` — 主界面空状态（Wawa 蛙插画）
- [ ] `03_main_list.png` — 主界面带多条 TODO 的列表（展示分类徽章、状态宝石）
- [ ] `04_todo_expanded.png` — 展开一个 research 任务查看 Agent 产出的 Markdown 报告
- [ ] `05_filter_search.png` — 过滤 + 搜索
- [ ] `06_settings_persona.png` — 设置页 Persona 身份画像
- [ ] `07_settings_llm.png` — 设置页 LLM 配置
- [ ] `08_settings_agent.png` — 设置页 Agent / 风险等级
- [ ] `09_settings_location.png` — 设置页当前位置 + 高德 Key
- [ ] `10_settings_obsidian.png` — 设置页 Obsidian 集成
- [ ] `11_settings_wechat_qr.png` — 微信扫码登录页
- [ ] `12_notification.png` — 到期提醒触发系统通知
- [ ] `13_pipeline_progress.png` — Agent 执行中的进度显示
- [ ] `14_wechat_receive.png` — 微信端收到 SATH 自动回复的关键信息

**截图原则**：
- macOS 用 `Cmd+Shift+4` 框选截图
- 每张带必要的内容（别空窗口）
- 不含任何真实 API Key、真实手机号、真实地址等敏感信息（可用假数据）

## 提交前自检清单

- [ ] 设计说明书与用户手册版本一致
- [ ] 源代码 PDF 生成且已核对前 30 / 后 30 页无重复
- [ ] 所有截图已按清单补齐
- [ ] 截图中无敏感信息
- [ ] 软件名称、版本号在所有文档中统一（当前：**SATH V1.0**）
- [ ] 技术栈描述准确（Tauri 2.0 + Rust + React + SQLite）
