2. Planner 规划流程

flowchart LR
  IN["原始输入\n+ Context\n+ Persona"] --> CHK{"库 > 12?"}
  CHK -->|是| ST1["Stage-1\nLLM 挑 ≤3 类目"]
  CHK -->|否| ST2
  ST1 --> ST2["Stage-2\nLLM → Pipeline JSON"]
  ST2 --> VAL["校验 agent\n剔除不存在的"]
  VAL --> DUE["本地 due_at\n覆盖 LLM 输出"]
  DUE --> CONF{"confidence\n< 0.75?"}
  CONF -->|是| REV["needs_review"]
  CONF -->|否| SAVE
  REV --> SAVE["写入 DB"]
  SAVE --> PIPE{"pipeline\n非空?"}
  PIPE -->|是| GO["spawn_pipeline()"]
  PIPE -->|否| END["仅存储"]

3. Executor DAG 调度

flowchart TB
  START["Pipeline 开始"] --> CYC{"依赖有环?"}
  CYC -->|是| LIN["回退线性"]
  CYC -->|否| LOOP
  LIN --> LOOP["主循环"]
  LOOP --> READY["找 ready steps\ndeps ⊆ completed"]
  READY --> EMPTY{"空?"}
  EMPTY -->|是| FIN
  EMPTY -->|否| SPAWN["并行 spawn\n所有 ready"]
  SPAWN --> WAIT["等待本批"]
  WAIT --> OK["成功 → completed\n增量写 output"]
  WAIT --> FAIL["失败 → 记录\n同批继续"]
  OK --> DONE{"全完成?"}
  DONE -->|否| LOOP
  DONE -->|是| FIN
  FAIL --> FIN["合并输出\n同步 Obsidian/飞书/微信\nemit pipeline_done"]

  style START fill:#fce4ec,stroke:#c62828
  style SPAWN fill:#e3f2fd,stroke:#1565c0
  style FIN fill:#e8f5e9,stroke:#4a7c59

4. 单 Step ReAct 循环

flowchart TB
  INIT["system: SOUL + 工具协议\nuser: persona + ctx + plan"] --> TURN["turn 1..max_turns"]
  TURN --> CHK1{"todo 还在?"}
  CHK1 -->|否| ABORT["中止"]
  CHK1 -->|是| NUDGE{"剩余 ≤2?"}
  NUDGE -->|是| WARN["追加 ⚠ final_answer"]
  NUDGE -->|否| CMP
  WARN --> CMP{"turn>4\nmsgs>8?"}
  CMP -->|是| CMPR["LLM 压缩\n中间历史"]
  CMP -->|否| CALL
  CMPR --> CALL["chat() → JSON"]
  CALL --> PARSE{"解析成功?"}
  PARSE -->|失败| PROSE{"像 prose?"}
  PROSE -->|是| IMPL["隐式 final_answer"]
  PROSE -->|否| RETRY["提示重试"]
  RETRY --> TURN
  PARSE -->|成功| RPT{"重复调用?"}
  RPT -->|连续2次| STUCK["中止: 打转"]
  RPT -->|否| ACT{"action"}
  ACT -->|final_answer| DONE["返回 markdown"]
  ACT -->|"web_search\nweb_fetch\nfs_*/shell_run"| TOOL["执行工具\n→ tool_result"]
  TOOL --> TURN

  style INIT fill:#f3e5f5,stroke:#7b1fa2
  style DONE fill:#e8f5e9,stroke:#4a7c59
  style ABORT fill:#ffcdd2,stroke:#c62828
  style STUCK fill:#ffcdd2,stroke:#c62828
